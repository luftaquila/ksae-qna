## Codebase Patterns
- Use `pyright` for typechecking (available at /opt/homebrew/bin/pyright)
- Directory structure: src/ for modules, data/raw/ for crawled data, data/processed/ for pipeline outputs
- Dependencies listed in requirements.txt (no pyproject.toml)
- KSAE board HTML: table rows have 7 td cells (лІҲнҳё, кө¬л¶„, м ңлӘ©, л“ұлЎқмһҗ, нҢҢмқј, мЎ°нҡҢмҲҳ, л“ұлЎқмқј)
- Pagination 'л§Ҳм§Җл§ү' button: `img[src*="block_next"]` inside an `<a>` tag; parse `page` param from href
- Post detail URL uses a `number` param (from the href) which differs from the display `лІҲнҳё` column
- Direct Python requests to www.ksae.org may time out; WebFetch tool works as alternative for analysis
- Use `from __future__ import annotations` for modern type hint syntax compatibility
- www.ksae.org has weak DH keys; must use `@SECLEVEL=1` SSL cipher config via custom HTTPAdapter
- Detail page table (`table.tblDef`): row 0 = title (class='bg'), row 1 = metadata (th/td pairs: мһ‘м„ұмһҗ, Date, Hits), row 2 = body (td class='con'), row 3+ = attachments (th class='th' with 'мІЁл¶ҖнҢҢмқј')
- Answers on KSAE Q&A are separate posts (not inline); reply posts have `img.reply[src*="bl_reply"]` in title cell
- Reply post body includes original question after `====== мӣҗ кёҖ ======` separator; split on this to extract answer only
- Metadata row has alternating `<th>` and `<td>` tags; use `find_all("td")` to get only values (index 0=author, 1=date, 2=hits)
- Chunk schema: `{post_id, category, title, date, url, chunk_index, text}` вҖ” saved to `data/processed/chunks.json`
- Token counting uses simple whitespace split (`str.split()`) for rough estimation
- FlagEmbedding has no type stubs; use `# type: ignore[import-untyped]` for pyright
- BGE-M3 model: `BGEM3FlagModel("BAAI/bge-m3", use_fp16=True)` вҖ” auto-detects GPU (CUDA/MPS/CPU)
- `model.encode()` returns dict with `dense_vecs` key containing numpy array of shape (batch, 1024)
- Qdrant point IDs: deterministic UUIDs via `uuid.uuid5(NAMESPACE_URL, f"{post_id}_{chunk_index}")` for idempotent upserts
- Qdrant models: import `Distance`, `PayloadSchemaType`, `PointStruct`, `VectorParams` from `qdrant_client.models`
- `upload_to_qdrant()` supports `recreate` flag for full vs incremental mode
- CLI entry point: `main.py` with click group; `python main.py` runs full pipeline, `python main.py <stage>` runs individual stage
- Global CLI options (--qdrant-url, --collection, --batch-size, --delay) are passed via `ctx.obj` dict
